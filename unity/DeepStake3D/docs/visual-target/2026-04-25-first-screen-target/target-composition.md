# Target Composition

## Core Read

The first screen should read in this order:
- player
- path / immediate route
- notice board
- nearby building frame
- quiet background edge

The player should be findable within 3 seconds.
The notice board should be understood as the first objective without relying on UI text.

## Landmark

The notice board should be the first clear landmark.

Required landmark language:
- strong silhouette
- dark frame or backing
- clear grounded base
- readable local contrast from current camera
- simple surrounding composition

Avoid:
- clutter around the landmark
- neon/debug emphasis
- tiny decorative detail that disappears at screenshot scale

## Buildings

Buildings should support the composition instead of collapsing into one dark mass.

Target building read:
- roof mass clearly separate from wall mass
- wall mass clearly separate from base/foundation
- entrance readable from camera
- windows readable as larger accents

Use larger readable shapes, not trim spam.
At least two first-screen buildings should feel materially different at a glance.

## Ground

Ground must stay continuous but visually secondary.

Target ground language:
- calm base terrain
- readable route to the notice board
- subtle local wear only where it helps composition
- no visible sheet/grid/ring dominance

Avoid:
- broad color carpets
- woodgrain/ring/grid dominance
- random noisy breakup

## Background

The top and right background should quietly hide the test-map edge.

Target background language:
- low berms
- fence rhythm
- tree line
- settlement edge blockers

These should read as low framing elements, not giant rounded blob hills.

## Player

The player zone should be quiet and readable.

Target player read:
- local ground under and behind player stays simple
- subtle contact/shadow support is allowed
- nearby background should not compete with the player

Do not solve player readability with flashy VFX or unrelated props.
